[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how data flows.
First, every query enters through the START node and immediately hits our security checkpoint node.
This checkpoint scrubs sensitive PII, evaluates the input for prompt injection keywords, and checks domain safety rules.
If a safety violation is detected, it routes immediately to the security violation handler to block the query and offer crisis support.
If the check passes, it routes to our lead orchestrator agent, which delegates the request to either the symptom analyzer or the clinical trial advisor sub-agents via AgentTool callouts.
Before any symptom summaries or clinical trials are displayed to the user, the flow pauses at the hitl_consent node, which is a Human-in-the-loop gate that requests explicit patient consent.
Only after the user replies with consent, does the final_response node render the formatted concierge guidance and end the session.
